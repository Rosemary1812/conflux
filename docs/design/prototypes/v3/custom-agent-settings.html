删除自建 Agent

你即将删除 Doc Reviewer（@doc-reviewer）。

该 Agent 仍出现在 1 个群聊的 roster 中：

  • · 群聊 产品文案 (1 个成员)

删除后，该 roster 会显示"已删除 Agent"。

该 Agent 的历史 run / 消息 / interaction 保留。

⚠️ 此操作不可恢复。

视图 1: 列表（list 状态）

显示所有自建 Agent，每行含头像 + displayName + alias + 能力 tag + 最后运行时间。空态显示引导文案。

  • 点击行任意位置 → edit.preview 详情
  • 点击右侧"编辑" → edit.editing 编辑面板（不经详情）
  • 不显示活跃/空闲状态（避免视觉拥挤）
  • 不显示所在群聊（避免视觉拥挤；V3.6 不强求在列表感知）
视图 2: 详情（edit.preview）

只读详情，4 个操作：重新生成 profile / 编辑 / 删除（带 Q6 防御）/ 关闭。

  • System Prompt 默认显示前 6 行，点 [展开] 看全部
  • 删除按钮 disabled + tooltip = precheck 失败态（active runs 存在）
  • precheck 通过 → 点删除弹视图 5 二次确认
视图 3: 编辑（edit.editing）

所有字段表单，capability tag 编辑器沿用 V3.5 .capability-tag 视觉。avatarKind=uploaded 真实接入走 /api/agents/:id/avatar。

  • alias 改名带 ⚠️ 提示：历史 roster 行 alias 保留
  • 重新生成 profile 不持久化 session，一次 LLM 调用回填
  • systemPrompt 实时字符计数
视图 4: 重新生成 LLM（edit.regenerating）

表单整体淡化 + 居中 spinner，覆盖在 main 区域上。完成后自动跳回编辑面板并预填新 draft（不进 regen-preview 状态，省一步）。

  • 复用 V3.2 buildPlannerPrompt + Anthropic / OpenAI Planner
  • 输入：当前 AgentSummary 字段 + 用户 instruction
  • 输出：agentDraftPartialSchema 解析的 patch，不落库
视图 5: 删除二次确认

precheck 通过后弹出，列出该 Agent 仍在的群聊 + roster 成员数。红色"确认删除"按钮高危样式。

  • 删除后 getConversationRoster 序列化降级"已删除 Agent" → 🤖
  • agent_skills 自动级联（Drizzle cascade）
  • 历史 agent_runs / agent_interactions / messages.agent_id 保留 dangle FK
avatarKind=uploaded 真实接入

V3.5 留降级到 🤖，V3.6 真实接入：

  • [更换头像] → 复用 /api/attachments/select?imageOnly=true（V1 已有，Windows OpenFileDialog）
  • 选中后存绝对路径到 agents.avatar_value（不复制 bytes）
  • 前端 <img src=/api/agents/:id/avatar> 流式获取
  • 服务端校验：路径必须在 process.cwd() 或 os.homedir() 子树，文件 ≤ 1MB
