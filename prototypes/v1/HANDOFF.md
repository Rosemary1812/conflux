# V1 前端 Handoff

这批 `prototypes/v1/*.html` 是 **正式前端开发前的结构与交互原型**，不是可直接复用到生产的页面代码。

## 这批原型已经定下来的内容

- 整体是三栏：左侧会话列表、中间消息区、右侧上下文面板。
- `新建聊天` 直接进入空白会话，首条消息里只允许 `@` 一个 Agent；发送后锁定，不允许在同一会话里改 Agent。
- `新建群聊` 直接进入空白会话，首条消息可 `@` 多个 Agent；一旦出现多 Agent，`Orchestrator` 自动加入。
- 底部输入区保留 4 个核心入口：图片、附件、当前工作区、发送/停止。
- 发送按钮和停止按钮是 **同一个按钮** 的两种状态。
- 右栏支持收起，且支持拖拽宽度。
- 左下角设置按钮打开设置层；在正式版里应实现为 modal / sheet，而不是业务路由跳转。

## 底部输入区控件语义

- 最左侧图标：上传图片。
- 中间图标：上传附件。
- 最长的工作区 pill：选择当前工作区，也就是 Agent 活动目录与产物输出目录。
- 最右侧按钮：发送；如果 Agent 正在运行，则切换为停止。

## 正式实现建议

- 用 Next.js App Router + 组件化方式重写，不直接复用 HTML。
- 建议拆分为：`AppShell`、`ConversationSidebar`、`MessageStream`、`Composer`、`ContextPanel`、`SettingsModal`。
- 当前原型主要提供：信息架构、入口位置、按钮语义、关键交互规则。
